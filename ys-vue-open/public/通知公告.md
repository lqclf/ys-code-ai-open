# Vue3 高通用全配置型 顶部消息通知管理组件 
## 组件开发核心要求
开发一个**Vue3 纯配置型通用消息通知组件**，适用于中后台管理系统顶部导航区（铃铛图标位），组件设计遵循「**零源码修改、全外部配置驱动**」原则，所有样式、交互、数据、展示规则均通过外部传入的配置项控制，具备极强的通用性、可扩展性、高自定义性，适配任意中后台管理系统的消息通知业务场景。

## 一、组件整体功能架构（完整闭环，无功能遗漏）
### ✅ 核心触发交互（2种触发方式，二选一/共存均可，可配置）
鼠标**移入**顶部铃铛图标 或 **点击**顶部铃铛图标，触发消息通知气泡弹窗展开；支持配置 移出图标/点击空白处 气泡自动收起，交互逻辑可配。

### ✅ 气泡弹窗固定布局结构（不可变结构，内容/样式全配置）
气泡弹窗分为【上、中、下】三个固定区域，区域内所有展示内容、样式、文案均支持外部配置：
1. 【顶部区域】：展示「消息分类/消息类型」+「对应消息数量」（例如：系统通知(5)、待办提醒(3)），类型分组、数量展示规则、文本样式全配置；
2. 【中部核心区域】：消息列表展示区，列表项渲染**业务指定的消息数据属性**，支持列表样式、滚动规则、空数据占位、未读/已读差异化样式全配置；
3. 【底部固定区域】：统一展示「前往通知中心」按钮，按钮的文案、样式、点击跳转逻辑全配置，无其他冗余内容。

### ✅ 消息列表项核心交互
1. 点击任意一条消息列表项 → 触发【自定义详情弹窗事件】，组件内部**只派发事件不封装弹窗**，弹窗的渲染、内容、样式、逻辑完全由父组件外部配置实现，组件仅传递当前点击的完整消息数据；
2. 消息项支持「未读/已读」两种状态，状态样式、状态切换规则全配置。

### ✅ 通知中心页面完整功能（组件集成，全配置驱动）
点击气泡弹窗底部「前往通知中心」按钮，跳转/打开【通知中心独立页面】，该页面为组件内置的完整子页面，页面内所有内容、数据、交互、样式**全部支持外部配置**，无硬编码：
1. 页面顶部固定3个标签页（Tab）：「全部消息」「未读消息」「已读消息」，Tab的文案、激活样式、切换逻辑、高亮颜色全配置；
2. Tab切换后，对应展示的表格数据**数据源完全独立配置**，支持不同Tab绑定不同接口/不同过滤条件的数据源；
3. 表格为自适应布局，表格的列配置、列宽、对齐方式、内容渲染规则全配置；
4. 表格右侧固定「操作栏」，仅包含「查看」按钮，点击按钮 → 触发和「点击消息列表项」**同一个自定义详情弹窗事件**，传递当前行完整消息数据，弹窗逻辑外部统一配置；
5. 通知中心页面的整体尺寸、弹窗规则、关闭逻辑全配置。

## 二、消息数据模型规范（必含基础属性 + 支持扩展，无侵入性）
### ✅ 基础必含数据属性（不可删减，组件内部适配解析）
所有消息数据源，无论业务扩展多少字段，**必须包含以下核心属性**，属性名固定、类型固定，保证组件通用性：
```ts
interface MessageItem {
  id: string | number; // 唯一标识，必传，组件内部做列表渲染/事件传参标识
  type: string; // 消息类型，必传，支持自定义枚举（如：system/notice/todo/alert），用于类型分组、类型样式匹配
  title: string; // 消息标题，必传，列表/表格中核心展示文案
  content: string; // 消息正文内容，必传，详情弹窗核心数据源
  sendTime: string | Date; // 消息发布时间，必传，支持字符串/日期格式，组件内部可配置格式化规则
  sendUser: string; // 消息发布人/来源方，必传（如：系统管理员/运营平台/张三）
  readStatus: 0 | 1; // 阅读状态 必传，0=未读，1=已读，组件内部适配状态样式、Tab筛选逻辑
}
```
### ✅ 数据扩展规则（核心通用特性）
组件完全支持在上述基础属性上，**无限制扩展任意业务字段**（如：linkUrl、businessId、status、source等），扩展字段会被组件完整透传，在「详情弹窗事件」「表格渲染」中均可正常获取使用，组件对扩展字段无任何过滤/修改，零侵入业务数据结构。

## 三、全维度完整配置项分类（核心重点，所有配置独立解耦，无硬编码，优先级：配置项 > 组件默认值）
组件所有功能、样式、交互、数据均通过 **一个顶层配置对象 props 传入**，配置项分为【显示配置、交互配置、数据配置、样式配置】四大类，四类配置完全解耦，可单独配置、组合配置，所有配置项均为「可选传」，未配置时组件提供合理默认值，配置后立即覆盖默认值，**无需修改组件一行源码**。
### ✅ 【显示配置】- 可视化相关全配置（粒度到最小元素，覆盖所有视觉需求）
控制组件所有节点的「尺寸、布局、图标、动画、文本、占位、显示/隐藏」，无样式硬编码，支持数值/字符串/枚举值配置，所有配置项如下：
1. 铃铛图标配置：图标样式/大小/颜色/hover颜色、未读消息小红点显示规则（是否显示、圆点位置、颜色、大小）、图标是否旋转/闪烁动画；
2. 气泡弹窗配置：弹窗宽度/高度（支持px/%/vh/vw）、最大高度（超出滚动）、弹窗定位偏移量（防止遮挡导航）、弹窗圆角/阴影/边框、是否显示遮罩层；
3. 气泡顶部区域：类型文本样式（字号/颜色/粗细）、数量角标样式（背景色/文字色/圆角/大小）、是否换行展示多类型、空消息时的占位文案；
4. 气泡消息列表：列表项高度/内边距、项之间的间距、未读/已读项的区分标识（左侧竖线/背景色/字体加粗）、列表项hover样式、空列表占位图/占位文案/占位样式；
5. 底部按钮：按钮宽度/高度、文案、字体样式、背景色/hover色/点击色、圆角、是否显示图标、图标位置；
6. 通知中心页面：页面整体宽度/高度、是否全屏展示、Tab栏的高度/间距、Tab激活态下划线样式、表格整体样式（边框/斑马纹/hover高亮）、表格列宽配置、操作栏按钮样式；
7. 通用显示规则：时间格式化规则（如：YYYY-MM-DD HH:mm:ss / 几分钟前 / 昨天）、超长文本处理（省略号/换行/tooltip）、消息类型对应图标配置（如：system=系统图标，todo=待办图标）；
8. 动画配置：气泡展开/收起动画（淡入淡出/下滑上滑）、动画时长、列表滚动过渡动画、Tab切换动画、表格加载动画。

### ✅ 【交互配置】- 行为逻辑相关全配置（覆盖所有操作场景，无固定交互逻辑）
控制组件所有「鼠标事件、点击事件、自动行为、状态切换、弹窗规则」，所有交互逻辑可配置，组件仅执行配置的逻辑，无硬编码交互，所有配置项如下：
1. 气泡触发规则：触发方式（仅移入/仅点击/移入+点击）、移入延迟展开时长、移出后延迟收起时长、点击空白处是否收起气泡、点击铃铛图标是否强制收起；
2. 消息项交互：点击消息项后是否自动标记为「已读」（readStatus=1）、是否支持双击消息项触发其他事件、hover消息项是否显示预览文案；
3. 核心自定义事件（组件核心能力，零业务耦合）：
   - `onMessageClick`: 点击「消息列表项/通知中心表格查看按钮」时触发，必传回调函数，入参为当前点击的完整`MessageItem`对象，外部通过该事件配置自定义详情弹窗，组件不封装弹窗；
   - `onToCenter`: 点击「前往通知中心」按钮时触发，可选回调函数，入参无，外部通过该事件配置「打开通知中心页面」的逻辑（路由跳转/弹窗打开/新标签页）；
4. 自动行为配置：是否开启消息列表「自动滚动播放」、滚动方向（上滑/下滑）、滚动速度、滚动暂停规则（hover列表时暂停/点击后暂停）、自动滚动循环次数；
5. 通知中心交互：Tab切换是否触发「重新加载数据源」、表格是否支持排序（按时间/发布人/类型）、表格是否支持分页（分页配置独立）、关闭通知中心页面是否触发回调；
6. 状态缓存配置：是否缓存消息的「已读/未读」状态、是否缓存用户最后一次查看的消息ID、是否在页面刷新后保留气泡展开状态。

### ✅ 【数据配置】- 数据源相关全配置（核心通用，适配所有数据场景，无硬编码数据逻辑）
控制组件所有「数据源来源、加载方式、过滤规则、分页、排序、联动逻辑」，组件不耦合任何接口/数据请求，纯数据消费型，所有数据相关逻辑全配置，所有配置项如下：
1. 核心数据源配置：
   - `messageList`: 气泡弹窗的消息列表数据源，类型为`Array<MessageItem>`，必传，外部传入（接口请求/本地模拟/全局状态），组件仅渲染，不修改源数据；
   - `tabDataSource`: 通知中心3个Tab的数据源，类型为`{ all: Array<MessageItem>, unread: Array<MessageItem>, read: Array<MessageItem> }`，必传，外部传入，不同Tab绑定不同数据源，组件仅根据Tab切换渲染对应数据，无硬编码过滤；
2. 数据加载方式：是否支持「懒加载」（滚动到底部加载更多消息）、懒加载阈值、每次加载条数、加载中占位文案/样式；
3. 数据过滤规则：是否过滤重复消息（根据id去重）、是否过滤过期消息（根据sendTime配置过期时间）、是否支持按「消息类型type」过滤气泡列表、是否支持按「发布人sendUser」筛选；
4. 分页配置：通知中心表格是否开启分页、每页条数、分页样式、分页触发回调、是否显示总条数/页码；
5. 排序配置：消息列表/表格默认排序规则（按sendTime倒序/正序、按id排序）、是否支持点击表头切换排序方式；
6. 空数据规则：无数据时的占位文案、是否显示「刷新数据」按钮、点击刷新按钮触发的回调函数（外部重新请求数据）。

### ✅ 【样式配置】- 主题/差异化样式全配置（多主题适配，粒度到类型/状态，无硬编码样式）
基于【显示配置】的基础上，做「全局主题、差异化样式、优先级样式」的配置，所有样式可独立配置，支持多主题切换，样式规则无冲突，所有配置项如下：
1. 全局主题配置：支持「浅色主题/深色主题/业务自定义主题」，配置项为主题枚举值，一键切换组件所有节点的底色、文字色、边框色、阴影色，适配管理系统的整体主题；
2. 优先级样式配置：组件所有样式支持「优先级权重」，配置的样式优先级 > 主题默认样式 > 组件内置样式，可单独覆盖某一个元素的样式，不影响其他元素；
3. 消息类型差异化样式：为不同的`type`值配置独立样式（如：system类型=蓝色、todo类型=橙色、alert类型=红色），包括：类型文本颜色、左侧标识颜色、图标样式、列表项背景色，支持自定义任意类型的样式映射；
4. 状态差异化样式：为「未读(readStatus=0)、已读(readStatus=1)」配置独立样式，包括：文字粗细、背景色、边框、是否显示未读红点，样式规则可完全自定义；
5. 自定义样式注入：支持传入`customClass`/`customStyle`，通过CSS类名/行内样式覆盖组件任意节点，无样式覆盖限制；
6. 响应式样式配置：支持根据屏幕尺寸（大屏/中屏/小屏）自动切换组件尺寸、字体大小、布局方式，适配不同分辨率的管理系统。

## 四、组件开发额外核心要求（保障通用性、易用性、可维护性）
1. 技术栈：纯 Vue3 开发，支持 `<script setup>` 语法糖，可兼容 Vue3 任意版本，无第三方UI库强依赖（如需图标可使用通用SVG/IconFont，配置项传入图标类名）；
2. 无侵入性：组件不修改外部传入的「数据源」「配置项」，所有数据均为只读，所有状态均在组件内部维护，不污染父组件/全局状态；
3. 兼容性：支持所有现代浏览器，适配中后台管理系统常用的分辨率（1920*1080、1366*768等），无兼容性样式/交互问题；
4. 可复用性：组件为独立.vue文件，无业务耦合代码，可直接在任意Vue3项目中引入，仅需传入配置项和数据源即可使用，无需二次开发；
5. 性能优化：消息列表/表格采用「虚拟滚动」（可配置是否开启），大数据量（1000+条）无卡顿，列表渲染采用key绑定id，避免重复渲染；
6. 容错处理：对数据源做容错校验（如：id为空、数组为空、字段类型错误），组件不会报错，而是显示空数据占位，增强健壮性；
7. 注释规范：组件内部代码有详细注释，配置项有完整的TS类型定义和注释，方便使用者阅读和配置。
